◆ scheduler_iterate()

uint32_t scheduler_iterate ( scheduler_context_t  context,
const uint64_t  timestamp 
)

Call this function repetitively.

Parameters
contextcame from scheduler_start
timestampin millisecond since program start or UNIX epoch
Returns
uint32_t as remaining milliseconds toward nearest task

It checks if there is any ready task to be called and updates the counter for each task. If the task is ready, this function also calls the handler. Remember to keep every task handler's execution time short as possible, or the program would experience overdue task or worst, the program hangs.

Example

#include <time.h>
// Sleep function in Windows is quite different
#if __has_include(<windows.h>)
#include <Windows.h>
#define SLEEP(t) Sleep(t)
#else
#include <unistd.h>
#define SLEEP(t) usleep(t*1000)
#endif
// ...
int main () {
scheduler_context_t my_scheduler = scheduler_start(my_tasks, 2);
struct timespec tms = {0};
// ...
while (1) {
timespec_get(&tms, TIME_UTC);
uint64_t timestamp = (tms.tv_sec * 1000) + (tms.tv_nsec / 1000000);
uint32_t remaining_milliseconds = scheduler_iterate(my_scheduler,
timestamp);
SLEEP(remaining_milliseconds);
}
// ...
}

◆ scheduler_start()

scheduler_context_t scheduler_start ( const struct scheduler_task *  task_array,
const unsigned int  task_count 
)

Initiate a single instance of scheduler.

Parameters
task_arraydefined as array of struct scheduler_task
task_countrepresents number of tasks in task_array
Returns
scheduler_context_t if task_array is valid, otherwise NULL

This function takes list of tasks from task_array parameter and then allocates a region of memory to take a count for each task. Keep in mind that the scheduler_start function won't do the iteration. scheduler_iterate does that.

Example

void first_task(uint32_t delta_millis) { ... }
void second_task(uint32_t delta_millis) { ... }
const struct scheduler_task my_tasks[] = {
{
.handler = &first_task,
.millisecond = 1000
},
{
.handler = &second_task,
.millisecond = 1500
}
};
int main() {
scheduler_context_t my_scheduler = scheduler_start(my_tasks, 2);
// ...
}

◆ scheduler_stop()

void scheduler_stop ( scheduler_context_t *  pointer_to_context)

Clean-up the started scheduler then NULLify the context.

Parameters
pointer_to_contextmust has ampersand prefix, as example &my_scheduler

Ensures the scheduler was cleaned-up from memory for safety. So accidental scheduler_iterate call won't cause segmentation fault. This happens by freeing the context and NULLify the context because scheduler_iterate will ignore NULL context. Therefore, the parameter is pointer_to_context, not regular context parameter. See the example for better explanation.

Example

// ...
int main() {
scheduler_context_t my_scheduler = scheduler_start(my_tasks, 2);
// ...
scheduler_stop(&my_scheduler);
return 0;
}
